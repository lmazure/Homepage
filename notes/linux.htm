consoles

  • in tty mode, use
    • Alt+F1, Alt+F2,... Alt+F6 to switch between virtual consoles
    • Alt+Right and Alt+Left to rotate through the virtual consoles
    • Alt+F7 to be back to the X session
  • in X mode, use
    Ctrl+Alt+F1    , ... Ctrl+Alt+F6 to switch between virtual consoles.

  • pour voir ou réparer la table des partitions (ne par détruire et recréer la même partition, son contenu sera perdu)
    • #> fdisk
     
  • current partition
    • /     192M
    /boot  23M
     
  • /etc/fstab defines where mount the file systems and if they must be automatically mounted during the boot
    • current file (user enables any user to mount):
    /dev/hda7               /                 ext2    defaults               1 1
    /dev/hda6               /boot             ext2    defaults               1 2
    /dev/hda8               swap              swap    defaults               0 0
    /dev/fd0                /mnt/floppy       auto    noauto,user            0 0
    /dev/cdrom              /mnt/cdrom        iso9660 noauto,user,ro,unhide  0 0
    /dev/hda1               /mnt/dos_c        vfat    defaults               0 0
    /dev/hda5               /mnt/dos_d        vfat    defaults               0 0
    none                    /proc             proc    defaults               0 0
    none                    /dev/pts          devpts  mode=0622              0 0
     
  • mount a DOS floppy if root
    • #> mount -o uid=500,gid=100 -t vfat /dev/fd0 /mnt/floppy
     
  • mount a DOS zip if root
    • #> mount -o uid=500,gid=100 -t vfat /dev/hdd4 /mnt/zip
     
  • mount the DOS partitions
    • #> mount -f vfat /dev/hda1 /mnt/dos_c
    #> mount -f vfat /dev/hda5 /mnt/dos_d
     
  • mount the CDROM
    • #> mount /mnt/cdrom
     
  • mount a CD burnt with Roxy Easy Creator
    #> mount -t udf /dev/cdrom /mnt/cdrom

  • unmount the CDROM and eject it
    • #> umount /mnt/cdrom
    #> eject /mnt/cdrom

  • when mounting a CDROM, use the -o exec option of mount to be able to execute files which are on the CDROM

network

  • display the IP address
    #> ifconfig -a

  • define the main parameters
    • hostname definition
      in /etc/sysconfig/network:
      HOSTNAME=foobar
      in /etc/hosts:
      127.0.0.1 foobar localhost.localdomain localhost
    • IP address definition
      in /etc/sysconfig/network-scripts/ifcfg-eth0:
      IPADDR=12.34.56.78
    • netmask definition
      in /etc/sysconfig/network-scripts/ifcfg-eth0:
      NETMASK=255.255.252.0
    • activate the hostname:
      #> hostname foobar
  • set the netmask
    • #> ifconfig eth0 netmask 255.0.0.0
  • add a default router
    • #> route add default gw 3.249.15.254

  • /etc/resolv.conf indicates host to resolve hostnames

  • trace the route of the packets
    #> traceroute

  • to export a new directory via NFS
    modify /etc/exports
    #> exportfs -ra

periphériques

  • list the SCSI devices currently attached
    #> cat /proc/scsi/scsi

X11
xdm
The default xdm configuration file is /usr/X11R6/lib/X11/xdm/xdm-config.
(/usr/X11R6/lib/X11/xdm is a symbolic link toward /etc/X11/xdm)
This can be modified by the -config file option on the command line.
DisplayManager.servers contains the specification of the X server or the name of a file containing this specification (the default is /etc/X11/xdm/Xservers).

gdm
gdm seems to be the Gnome version of xdm.

/var/gdm/:0.log is the log file for display 0 (only root can access this).

/etc/X11/gdm/gdm.conf is the configuration file.
0=/usr/bin/X11/X -bpp 16 o start the X server with 16 bits per pixels.

XFree86
XFree86 supports the following command line options:
-bpp n        sets the number of bits per pixels
-showconfig   shows the current configuration
and the following shortcuts
Ctrl+Alt+Backspace    kills the X server
Ctrl+Alt+KeypadPlus   switches to the next video mode (does not work #TBD#)
Ctrl+Alt+KeypadMinus  switches to the previous video mode (does not work #TBD#)
the configuration file is /usr/X11R6/lib/X11/XF86Config (symbolic link to /etc/X11/XF86Config).

xvidtune can be used to switch to next next/previous video mode, but it does not work (#TBD#).

enable the Xinerama mode
Section "ServerFlags"
    Option "Xinerama" "True"
EndSection
