In order to help code reviews, this script finds out all Unix 98 system calls and header files and will create a HTML link toward the OpenGroup man pages. This last point is not as much useful as the simple fact to highlight system calls.
The new version (May 12, 2001) is much faster than the previous one. Slowness was the main problem. Now, I may or may not fix the issues described below.

Issues:

  1. Some entries have been forgotten on the OpenGroup index pages. Since I used them as a starting point, there are also missing here. Current known missing data is:
    • all socket macros listed in xns/syssocket.h.html
    • in fact all macros...
    • and all constants... (anyway, the current script will not be able to recognize them)
  2. Some entries (e.g. write) are present in both system interface and network interface.
  3. routines preceded by ".", "::" or "->" (i.e. class/object methods) should be ignored
Notes:
This script is tuned for my  personal working environment: it assumes that Netscape is available and running.

while (defined $ARGV[0]) {

  my $datafile = $ARGV[0];
  my $htmlfile = $datafile;
  $htmlfile =~ s+^.*/(.*)$+$1+o;
  $htmlfile = "/tmp/".$htmlfile.".html";

  open(DATA,"< ".$datafile) or die("cannot open file $datafile ($!) --");
  open(HTML,"> ".$htmlfile) or die("cannot open file $htmlfile ($!) --");

  my $line = <DATA>;
  $line =~ s/</\&lt;/go;
  $line =~ s/>/\&gt;/go;
  $line =~ s/\t/        /go;
  $line =~ s!^( *# *include *)&lt;([a-z/]+\.h)&gt;!$1\&lt;<A HREF="$records_h{$2}">$2</A>\&gt;!gmo;
  $line =~ s!(\b[a-zA-Z]\w*)( *\()!<A HREF="$records_r{$1}">$1</A>$2!go;
  $line =~ s!<A HREF="">([^ ]*)</A>!$1!go;

  print HTML "<HTML><HEAD><TITLE>",
             $datafile,
             "</TITLE><BASE TARGET='_blank'></HEAD><BODY BGCOLOR='white'><PRE>",
             $line,
             "</PRE></BODY></HTML>";

  close(DATA);
  close(HTML);

  my @cmds = ("netscape", "-remote", "file:".$htmlfile);
  system(@cmds) == 0 or die("failed to communicate with Netscape --");
